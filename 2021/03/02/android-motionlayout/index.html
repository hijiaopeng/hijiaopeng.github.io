（Android）MotionLayout管理运动和微件动画

Android
Android
发布日期:   2021-03-02
更新日期:   2021-03-02
文章字数:   1.4k
阅读时长:   6 分
阅读次数:  

概述

MotionLayout是一种布局类型，是ConstraintLayout 库的一部分，实现了运动形式的动画，缩小了布局转换与复杂运动处理之间的差距，同时在属性动画框架和 CoordinatorLayout之间提供了各种功能。

MotionLayout完全可以使用xml文件来描述任何转换和动画，无论多么复杂。值得注意的是，MotionLayout 仅适用于其直接子级，不支持嵌套布局层次结构或 Activity 转换。

项目中要做一个仿携程的城市交换动画，一开始使用属性动画写起来很是麻烦，后来就用了MotionLayout，虽然还是有些瑕疵，不过逻辑处理上清晰了很多。

声明依赖项

implementation 'androidx.constraintlayout:constraintlayout:2.0.4'

引入版本需要在2.0以上，旧版本不支持。

绘制页面布局xml

<androidx.constraintlayout.motion.widget.MotionLayout
        android:id="@+id/motionLayout"
        android:layout_width="match_parent"
        android:layout_height="wrap_content"
        app:layoutDescription="@xml/scene1"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintTop_toTopOf="parent"
        app:showPaths="true">

        <TextView
            android:id="@+id/button"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:gravity="center"
            android:text="北京站"
            android:textSize="16sp" />

        <TextView
            android:id="@+id/button1"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:gravity="center"
            android:text="上海站"
            android:textSize="16sp"
            app:layout_constraintEnd_toEndOf="parent" />

</androidx.constraintlayout.motion.widget.MotionLayout>

app:layoutDescription：

引用的是一个MotionScene，它是一个xml资源文件，用来描述所有的运动。为了将布局信息与运动描述分开，每个MotionLayout都要引用一个单独的MotionScene。注意，MotionScene 中的定义优先于 MotionLayout 中的任何类似定义。

app:showPaths：

表示在运动进行时是否显示运动路径。此属性的默认值为 false。

app:applyMotionScene="boolean：

表示是否应用 MotionScene。此属性的默认值为 true。

app:progress="float：

可让您明确指定转换进度。您可以使用从 0（转换开始）到 1（转换结束）之间的任意浮点值。

app:currentState="reference：

可让您指定具体的 ConstraintSet。

app:motionDebug：

可让您显示与运动有关的其他调试信息。可能的值为“SHOW_PROGRESS”、“SHOW_PATH”或“SHOW_ALL”。

配置MotionScene描述文件

在res下创建xml包，新建scene.xml描述文件。

<?xml version="1.0" encoding="utf-8"?>
<MotionScene xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:motion="http://schemas.android.com/apk/res-auto">

    <!--运动的基本定义
        motion:constraintSetStart 和 motion:constraintSetEnd：运动的端点。
        这些端点在 MotionScene 后面的 <ConstraintSet> 元素中定义。
        motion:duration：完成运动所需的毫秒数。
    -->
    <Transition
        motion:constraintSetEnd="@+id/end"
        motion:constraintSetStart="@+id/start"
        motion:duration="500" >

        <!--轻触控制运动
            motion:touchAnchorId：可以滑动并拖动的视图。
            motion:touchAnchorSide：表示我们从右侧拖动视图。
            motion:dragDirection：拖动的进度方向。例如，motion:dragDirection="dragRight" 表示当您向右拖动时，进度会增加。
        -->
        <OnSwipe
            motion:touchAnchorId="@+id/button"
            motion:touchAnchorSide="right"
            motion:dragDirection="dragRight" />

    </Transition>

    <!--定义描述运动的各种限制条件的位置。-->
    <ConstraintSet
        android:id="@+id/start">

        <!--android:id：需要和布局文件中MotionLayout里面子组件的id一致，一一对应。-->
        <Constraint
            android:id="@+id/button"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:layout_marginStart="8dp"
            motion:layout_constraintBottom_toBottomOf="parent"
            motion:layout_constraintStart_toStartOf="parent"
            motion:layout_constraintTop_toTopOf="parent" />

        <Constraint
            android:id="@+id/button1"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:layout_marginEnd="8dp"
            motion:layout_constraintBottom_toBottomOf="parent"
            motion:layout_constraintEnd_toEndOf="parent"
            motion:layout_constraintTop_toTopOf="parent"/>

        <!--自定义属性，为不仅仅与位置相关的属性或 View 属性指定转换。
            这个标签也可以用在Constraint标签里面
            motion:attributeName：是必需属性，并且必须与具有 getter 和 setter 方法的对象匹配。getter 和 setter 与特定模式非常匹配。
            此外，您必须提供的另一个属性基于值类型。从以下支持的类型中选择：
            motion:customColorValue 适用于颜色
            motion:customIntegerValue 适用于整数
            motion:customFloatValue 适用于浮点值
            motion:customStringValue 适用于字符串
            motion:customDimension 适用于尺寸
            motion:customBoolean 适用于布尔值
            注意，在指定自定义属性时，您必须定义开始和结束 <ConstraintSet> 元素中的端点值。
        -->
        <CustomAttribute
            motion:attributeName="backgroundColor"
            motion:customColorValue="#D81B60"/>

    </ConstraintSet>

    <ConstraintSet
        android:id="@+id/end">
        <Constraint
            android:id="@+id/button"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:layout_marginEnd="8dp"
            motion:layout_constraintBottom_toBottomOf="parent"
            motion:layout_constraintEnd_toEndOf="parent"
            motion:layout_constraintTop_toTopOf="parent" />

        <Constraint
            android:id="@+id/button1"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:layout_marginStart="8dp"
            motion:layout_constraintBottom_toBottomOf="parent"
            motion:layout_constraintStart_toStartOf="parent"
            motion:layout_constraintTop_toTopOf="parent" />
    </ConstraintSet>

</MotionScene>

KeyFrameSet关键帧

这个标签可以改变我们动画过程中某个关键帧的位置以及状态信息。

<Transition
        app:constraintSetEnd="@id/end"
        app:constraintSetStart="@id/start">

        <KeyFrameSet>
          
            <KeyPosition
                app:framePosition="50"
                app:keyPositionType="parentRelative"
                app:motionTarget="@id/windmill"
                app:percentY="0.5" />
          
            <!--前半段的动画效果：逆时针旋转一圈，同时放大一倍-->
            <KeyAttribute
                app:framePosition="50"
                app:motionTarget="@id/windmill"
                android:rotation="-360"
                android:scaleX="2.0"
                android:scaleY="2.0" />
          
            <!--后半段的动画效果：逆时针旋转一圈，同时变回原样-->
            <KeyAttribute
                app:framePosition="100"
                app:motionTarget="@id/windmill"
                android:rotation="-720" />
          
            <!--延迟动画——0-85过程中将透明度一直维持在0.0-->
            <KeyAttribute
                app:framePosition="85"
                app:motionTarget="@id/tipText"
                android:alpha="0.0" />

        </KeyFrameSet>

    </Transition>

KeyPosition和KeyAttribute：用来设置动画某个位置的关键帧，进而为某段动画指定所期望的效果。

KeyPosition用于指定动画某个关键帧的位置信息。

KeyAttribute 则用来描述动画某关键帧的属性配置（如：透明度、缩放、旋转等）。

页面中使用

//开始动画，起始点和终点
motionLayout?.transitionToStart()
motionLayout?.transitionToEnd()

motionLayout?.setTransitionListener(object : MotionLayout.TransitionListener {
            override fun onTransitionCompleted(p0: MotionLayout?, p1: Int) {
              //    动画完全执行完毕
            }

            override fun onTransitionStarted(p0: MotionLayout?, p1: Int, p2: Int) {
              //动画开始执行
            }

            override fun onTransitionChange(p0: MotionLayout?, p1: Int, p2: Int, p3: Float) {
              //动画改变过程
            }

            override fun onTransitionTrigger(p0: MotionLayout?, p1: Int, p2: Boolean, p3: Float) {
              //动画触发时监听
            }
        })

更多事例可以查看MotionLayout 示例，官方已经写好了，以提供参考。

结语

MotionLayout解决了我们绘制动画的复杂性，减少了相当多的计算绘制代码，但是也不能说就一定要用它，视情况和需求而定。
