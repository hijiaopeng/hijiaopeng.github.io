Android 使用华为的扫描功能

前言

关于扫描二维码功能，我们最多使用的就是ZXing了，有位同事说，你可以再尝试一下华为出的Scan Kit，据说两米开外都能扫描到，很是强大。对于这么NB的功能技术，我也忍不住体验了一把，最后感觉是：真的香…

开发准备

华为HMS Core官方文档，统一扫码服务：https://developer.huawei.com/consumer/cn/doc/development/HMSCore-Guides-V5/service-introduction-0000001050041994-V5

集成SDK

在项目的build.gradle文件中添加：

buildscript {
    repositories {
        ...
        // 配置HMS Core SDK的Maven仓地址。
        maven {url 'https://developer.huawei.com/repo/'}
    }
}

allprojects {
    repositories {
        ...
        // 配置HMS Core SDK的Maven仓地址。
        maven {url 'https://developer.huawei.com/repo/'}
    }
}

在Module的build.gradle文件中添加：

implementation 'com.huawei.hms:scanplus:1.3.2.300'

注意：请自行查找最新的版本号。

当然，为了适配到其他机型，我们选择的是plus版本。

配置混淆

找到proguard-rules.pro文件添加：

-ignorewarnings
-keepattributes *Annotation*
-keepattributes Exceptions
-keepattributes InnerClasses
-keepattributes Signature
-keepattributes SourceFile,LineNumberTable
-keep class com.huawei.hianalytics.**{*;}
-keep class com.huawei.updatesdk.**{*;}
-keep class com.huawei.hms.**{*;}

如果还使用了AndResGuard，需要在项目的“build.gradle”文件中加入AndResGuard允许清单：

"R.string.hms*",

"R.string.agc*"

指定权限

在AndroidManifest.xml文件中先指定所需的权限：

<!--相机权限-->
<uses-permission android:name="android.permission.CAMERA" />
<!--读写文件权限，在需要识别本地相册图片功能时添加-->
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />

后续在编写功能代码时，不要忘记动态申请权限。

功能实现

默认

极其简单的实现扫码功能，UI页面使用的SDK里面的，需要开启硬件加速，在AndroidManifest.xml文件中添加：

<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android">

    <application
        ...
                android:hardwareAccelerated="true"
                >
       
                ...

    </application>

</manifest>

配置扫码参数：

//Scan Kit默认支持13种码制式，开发者也可以指定Scan Kit只扫描特定的码制式以提高扫码速度
//这里“QRCODE_SCAN_TYPE”和“DATAMATRIX_SCAN_TYPE”表示只扫描QR和Data Matrix的码
val options = HmsScanAnalyzerOptions.Creator().setHmsScanTypes(HmsScan.QRCODE_SCAN_TYPE , HmsScan.DATAMATRIX_SCAN_TYPE).create()

启动扫码页面：

//REQUEST_CODE_SCAN_ONE：对应“onActivityResult”方法中的“requestCode”参数，用于判断“onActivityResult”调用是否来自Scan Kit扫码结果回调
//如果没有指定检测特定的码制式，此处的“options”可以置为“null”，表示默认检测Scan Kit支持的码制式。
ScanUtil.startScan(this, REQUEST_CODE_SCAN_ONE, options)

获取扫码结果：

override fun onActivityResult(requestCode:Int, resultCode:Int, data: Intent?) {
        super.onActivityResult(requestCode, resultCode, data)
        if (resultCode != RESULT_OK || data == null) {
            return
        }
        if (requestCode == REQUEST_CODE_SCAN_ONE) {
            //导入图片扫描返回结果
            var obj = data?.getParcelableExtra(ScanUtil.RESULT) as HmsScan?
            if (obj != null) {
                //展示解码结果,获取条码原始的全部码值信息
                obj.originalValue
            }
        }
    }
自定义UI

在布局文件中添加全屏的FrameLayout：

<?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/parentView"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <FrameLayout
        android:id="@+id/rim"
        android:layout_width="match_parent"
        android:layout_height="match_parent"/>
        
</androidx.constraintlayout.widget.ConstraintLayout>

代码创建RemoteView：

//初始化RemoteView，并通过如下方法设置参数:setContext()（必选）传入context、setBoundingBox()设置扫描区域、setFormat()设置识别码制式，
//设置完毕调用build()方法完成创建。通过setContinuouslyScan（可选）方法设置非连续扫码模式。
val remoteView: RemoteView by lazy {
    //设置扫码识别区域，您可以按照需求调整参数
    val dm = resources.displayMetrics
    val density = dm.density
    val mScreenWidth = resources.displayMetrics.widthPixels
    val mScreenHeight = resources.displayMetrics.heightPixels
    //当前demo扫码框的宽高是300dp
    val SCAN_FRAME_SIZE = 300
    val scanFrameSize = (SCAN_FRAME_SIZE * density).toInt()
    val rect = Rect()
    rect.left = mScreenWidth / 2 - scanFrameSize / 2
    rect.right = mScreenWidth / 2 + scanFrameSize / 2
    rect.top = mScreenHeight / 2 - scanFrameSize / 2
    rect.bottom = mScreenHeight / 2 + scanFrameSize / 2
    RemoteView.Builder().setContext(this).setBoundingBox(rect).setContinuouslyScan(false)
        .setFormat(
            HmsScan.QRCODE_SCAN_TYPE, HmsScan.DATAMATRIX_SCAN_TYPE
        ).build()
}

在onCreate()里面添加：

//将自定义view加载到activity
remoteView.onCreate(savedInstanceState)
val params = FrameLayout.LayoutParams(
    LinearLayout.LayoutParams.MATCH_PARENT,
    LinearLayout.LayoutParams.MATCH_PARENT
)
binding.rim.addView(remoteView, params)

注意：View的获取建议使用ViewBinding。

获取扫描结果：

//识别结果回调事件订阅
remoteView.setOnResultCallback {
      //获取到扫码结果HmsScan
      it.forEach { hs->
         Log.e("JP", "onCreate: ${hs.originalValue}")
      }
}

扫描结果会返回一个HmsScan数组，因为它支持一次扫描多个二维码，把每个扫描结果都会获取到，解决了误扫的问题。

其次，还需要管理一下生命周期：

override fun onStart() {
    super.onStart()
    //侦听activity的onStart
    remoteView.onStart()
}
override fun onResume() {
    super.onResume()
    //侦听activity的onResume
    remoteView.onResume()
}
override fun onPause() {
    super.onPause()
    //侦听activity的onPause
    remoteView.onPause()
}
override fun onStop() {
    super.onStop()
    //侦听activity的onStop
    remoteView.onStop()
}
override fun onDestroy() {
    super.onDestroy()
    //侦听activity的onDestroy
    remoteView.onDestroy()
}

只有这个地方，如果使用lifecycle就更好了。

结语
